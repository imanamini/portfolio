QA was the firewall
for every library release.

A one-line change inside ngx-button could ripple through every product. Without automation, the only net was manual regression across every consumer app — on every release.

Design decision

The demo app is the seam.

Each library gets a standalone demo page at /package/<name>. Playwright navigates to that URL — no build step, no published package, no mocks.

Why it matters

Live source, not dist/.

Demo imports libraries directly from source. A change shows up instantly — no publish loop to test.

04 · Test Architecture

Two layers. Different jobs.

UNIT Karma + Jasmine

Contract verification.

Lives next to the component. Runs headless. Cheap. Fast. Asserts what the component does, not how it looks.

  • Default & variant input values
  • Output signals & event emission
  • OnPush change-detection behavior
  • Class / style bindings per state
  • 100% code coverage target
E2E Playwright · Chromium

Visual + interaction contract.

Navigates to the demo page. Measures computed styles, captures snapshots, clicks things. Catches regressions QA would catch — but in seconds, not days.

  • Computed CSS per variant (size, color, state)
  • Animation contracts (duration, easing, name)
  • Style & visual snapshots per OS/browser
  • Responsive viewports (mobile / tablet / desktop)
  • Real user interactions against the live demo

05 · Demo — A Real E2E Test

We don't check that it rendered.
We check the exact computed pixel.

playwright ngx-spinner.spec.ts — Size Contract 
const sizeSpec = {
  sm: { size: '16px', stroke: '3px' },
  md: { size: '20px', stroke: '4px' },
  lg: { size: '24px', stroke: '4px' },
  xl: { size: '32px', stroke: '4px' },
};

for (const [name, expected] of Object.entries(sizeSpec)) {
  test(`"${name}" → ${expected.size}`, async ({ page }) => {
    const el = page.locator(`ngx-spinner[size="${name}"] .spinner`);
    const actual = await el.evaluate(el => {
      const s = getComputedStyle(el);
      return {
        spinnerSize: s.getPropertyValue('--spinner-size').trim(),
        strokeWidth: s.getPropertyValue('--stroke-width').trim(),
      };
    });
    expect(actual.spinnerSize).toBe(expected.size);   // ← exact
    expect(actual.strokeWidth).toBe(expected.stroke); // ← exact
  });
}
What this catches

Design token drift.

Someone bumps --spinner-size from 16→15px "to fit"? Fails in CI, blocks the PR.

What humans miss

Four sizes × two colors = 8 variants.

No QA is going to eyeball every variant every release. Playwright does, silently.

Pattern

This pattern repeats across 38 specs — every component has a Size/Color/State contract block.

09 · Lessons Learned

Every failure became a rule.

01 · MISSING COVERAGE

"If the demo doesn't show it, add it."

Claude would skip inputs the demo didn't expose. Rule: modify the demo to expose every variant. No skipping allowed.

02 · ONPUSH SURPRISES

Setting an input doesn't render it.

OnPush components need an explicit detectChanges(). We now test that contract and the negative case.

03 · PROTOCOL

Learning is mandatory.

Every new rule is applied to all existing specs and documented in TESTING.md + the slash-command. Failures compound into knowledge.
